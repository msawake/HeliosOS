Delegation Helpers

delegate_to_subagent()

Orchestrator → doer delegation with automatic retry loop. On each failure, appends error context to prompt. After max_attempts exhausted, sets circuit_broken = True and needs_attention = True.

delegate_parallel()

Delegates multiple tasks to different agents concurrently via asyncio.gather(). Takes a list of (agent_id, prompt, metadata) tuples. Each delegation goes through delegate_to_subagent with its own retry loop.

Six hooks in src/core/hooks.py form the defense-in-depth governance layer. Every tool call passes through the HookChain before and after execution.

Hook Details

1. AuditLogger

Append-only audit log. Hashes tool inputs with SHA-256 for integrity. Buffered writes to PostgreSQL audit_log table. Logs 12 fields per entry: timestamp, agent_id, agent_type, department, tier, session_id, hook_event, tool_name, tool_input_hash, decision, reasoning, model.

2. RateLimiter

Two-level rate limiting to prevent runaway loops:

LimitValueWindow
Per-session tool calls100Session lifetime
Per-minute tool calls3060-second sliding window

3. AuthChecker

Enforces least-privilege tool access. Three layers of checks:

Tool permission matching: Exact match or wildcard prefix (e.g. mcp__google-workspace__*)

14 blocked bash patterns:

rm -rf /          rm -rf .          DROP TABLE        DROP DATABASE
TRUNCATE TABLE    curl ... | sh     curl ... | bash   wget ... | sh
mkfs.             dd if=            fork bomb         chmod -R 777 /
> /dev/sd          shutdown/reboot/init 0

6 sensitive tools with restrictions:

ToolRestriction
send_gmail_messageRequires compliance check
send_messageRequires compliance check
create_eventMax tier 2 (no workers)
share_drive_fileRequires audit logging
set_drive_file_permissionsRequires audit logging
transfer_drive_ownershipRequires HITL approval

Tier-3 agent block: Workers (tier 3) cannot use the Agent tool — no sub-spawning.

4. CostTracker

Tracks token consumption and USD cost per session and globally. Enforces $50/session budget.

ModelInput ($/1M tokens)Output ($/1M tokens)
claude-opus-4-6$5.00$25.00
claude-sonnet-4-5-20250514$3.00$15.00
claude-haiku-4-5-20251001$1.00$5.00

5. ComplianceChecker

Validates agent outputs before external-facing actions. Checks for:

5 blocklist regex patterns: guaranteed returns/profits, "will never fail", exposed credentials, SSN references, credit card numbers.

3 required disclaimers: financial_advice, legal_opinion, medical content types.

Mass email check: >50 recipients triggers ASK_HUMAN.

KnowledgeBase

Company knowledge store. Production: PostgreSQL + Pinecone vector search.

KnowledgeEntry: 10 fields — id, category (policy|procedure|decision|faq|technical), title, content, tags, created_by, created_at, updated_at, department, references.

Search: Current implementation uses keyword matching with scoring (title=10, content=5, tag=3). Production uses Pinecone vector similarity with 1536-dim embeddings.

Methods: add(), search(), get(), add_decision_precedent()

MetricsStore

Business metrics and KPI tracking. Production: Prometheus/Datadog + PostgreSQL.

MetricPoint: 5 fields — name, value, timestamp, department, tags.

Gauge pattern: _gauges dict maintains current value per metric name. _metrics list stores full time-series history.

Methods: record(), increment(), get_current(), query(), get_dashboard()

TaskGraphBuilder — Fluent API

workflow = (
    TaskGraphBuilder("Lead Qualification", "operational")
    .task(name="research",    agent="sales-researcher", budget_tokens=40_000)
    .task(name="score",       agent="sales-scorer",     blocked_by=["research"])
    .task(name="enrich_crm",  agent="sales-ops",        blocked_by=["score"])
    .task(name="route",       agent="sales-lead",       blocked_by=["score"])
    .task(name="outreach",    agent="sales-sdr",        blocked_by=["route"])
    .build()
)

The builder automatically resolves blocked_by names to task IDs and populates reverse blocks references.

WorkflowEngine.tick() Flow

1
Find Ready

→
2
Dispatch
Set task IN_PROGRESS, invoke agent via AgentInvoker
→
3
Execute
Agent runs with TaskMetadata, result stored on task
→
4
Check
If all tasks complete → workflow COMPLETED. If failure & retries left → back to PENDING

Bootstrap Sequence

src/bootstrap.py — The "power switch" for any ForgeOS company. Wires together all platform components and company packages.

Operating Modes

Shadow

Agents run but all outputs go to review queue. No external actions. Good for initial deployment validation.

Supervised

Agents are primary operators. Humans review before any external-facing actions (emails, file sharing, calendar events).

Autonomous

Agents operate independently. Humans review daily summaries and handle escalations only.

CLI Usage

python -m src.bootstrap                                    # Default: LeadForge, supervised
python -m src.bootstrap --company dealforge                 # Boot DealForge AI
python -m src.bootstrap --mode autonomous --dashboard       # Autonomous + web dashboard
python -m src.bootstrap --company leadforge --demo          # Run LeadForge demo scenario
python -m src.bootstrap --mode shadow --loop                # Shadow mode with main loop

Main Loop (per tick, every 30s)

1. Workflow Tick

Engine dispatches ready tasks across all running workflows

2. Heartbeat Metrics

Record tick_count and active_workflows gauge

3. HITL Check

Count pending approvals, record to metrics

4. Sleep

Await 30 seconds before next tick
