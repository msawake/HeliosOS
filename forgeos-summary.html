Stack adapter interface details

Every stack implements the AgentStackAdapter ABC from stacks/base.py:

  • create_agent(agent_def) → str — Provision an agent from definition, return agent_id
  • invoke(agent_id, prompt, context) → AgentResult — Run a single invocation
  • start_loop(agent_id) — Start persistent loop for always-on / autonomous agents
  • stop(agent_id) — Graceful shutdown
  • get_status(agent_id) → AgentStatus — IDLE, RUNNING, PAUSED, STOPPED, FAILED, COMPLETED
  • scaffold_files(agent_def) → dict[path, content] — Generate template file tree for the stack

Shared data types: AgentDefinition, AgentResult, LLMConfig, ExecutionType, OwnershipType

Execution Types

Five lifecycle modes define how agents run. The PlatformExecutor wires each type at deploy time.

∞ Always-On

ALWAYS_ON

Long-running daemon. Calls adapter.start_loop() at deploy — agent runs continuously with configurable sleep interval (default 60s, OpenClaw 900s).

🕓 Scheduled

SCHEDULED

Cron-driven. Registers job via scheduler.add_job(cron_expr, callback). Supports "every Nm", "every Nh", and "*/N * * * *" syntax.

⚡ Event-Driven

EVENT_DRIVEN

Pub/sub reactive. Subscribes to named events via event_bus.subscribe(). Fired events trigger parallel invocations with event payload as context.

✋ Reflex

REFLEX

On-demand only. No wiring at deploy time. Invoked explicitly via API or parent agent delegation. The simplest execution model.

🤖 Autonomous

AUTONOMOUS

Goal-directed loop. Spawns asyncio.Task that invokes repeatedly (up to max_iterations, default 50) until COMPLETED or FAILED.

Hook decisions & HITL flow

Each hook returns one of three decisions:

  • ALLOW — Proceed with the tool call
  • BLOCK — Reject immediately with reason
  • ASK_HUMAN — Pause agent, create HITL approval request, send Slack notification with dashboard link. Agent resumes when human approves or denies.

The HookChain orchestrator runs pre_tool_use() (checks 1-3, 5) before execution and post_tool_use() (checks 4, 6-7) after. If any pre-check returns BLOCK, the tool call never executes.

Data Flows

How agents are deployed, invoked, triggered by events, and run on schedule.

1
AgentDefinition created
Name, stack, execution_type, ownership, LLM config, tools, schedule/triggers defined via dashboard or API.
2
PlatformExecutor.deploy()
Central dispatcher receives the definition and orchestrates the full deployment pipeline.
3
Scaffold files generated
Stack adapter's scaffold_files() produces template files. Written to agents/{personal|shared}/{name}/.
4
Registry & adapter registration
registry.register() stores agent metadata. adapter.create_agent() provisions within the stack.
5
Execution wiring
_wire_execution() activates based on type: start_loop (always-on), add_job (scheduled), subscribe (event-driven), create_task (autonomous).
1
API or parent delegation
POST /api/platform/agents/{id}/invoke with prompt and optional context.
2
Registry lookup
PlatformExecutor retrieves AgentDefinition from registry, resolves the correct stack adapter.
3
Stack adapter invoke
adapter.invoke() routes to LLMRouter.chat() with system prompt + user prompt.
4
LLM provider call
LLMRouter dispatches to Anthropic or OpenAI based on model prefix. Falls back to simulation if no API key.
5
AgentResult returned
Status, output text, tool calls, tokens used, and elapsed time propagated back to caller.
1
Event fired
POST /api/platform/events with event name (e.g. "email.received") and payload. Or fired programmatically.
2
Subscriber lookup
EventBus finds all agents subscribed to the event name. Event recorded in history (max 1000).
3
Parallel dispatch
All subscriber callbacks executed concurrently via asyncio.gather().
4
Agent invocation
Each callback invokes its agent with the event payload as context. Failures are caught and logged per-callback.
5
Notified list returned
API returns the list of agent IDs that were notified for the event.
1
Job registered
Agent deployed with execution_type=SCHEDULED. Cron expression parsed (e.g. "every 15m" → 900s interval).
2
Background task spawned
asyncio.Task created. Sleeps for the parsed interval between invocations.
3
Callback executes
On wake: executor.invoke(agent_id, "Scheduled run: ..."). Timestamp recorded as last_run.
4
Loop continues
Sleeps again for the interval. Continues until scheduler.remove_job() or platform shutdown.

Multi-tenancy architecture
  • Authentication: Firebase Auth (JWT) for dashboard users, API key auth for programmatic access
  • RBAC: Three roles — Admin (configure agents), Operator (approve HITL, monitor), Viewer (read-only)
  • Database isolation: PostgreSQL Row-Level Security. DatabaseClient.tenant(id) sets SET app.current_tenant session variable. All queries auto-filtered.
  • Billing: Stripe subscriptions with metered overage. Webhook handling for subscription changes, payment failures. Customer self-service portal.
  • Usage tracking: UsageEnforcer checks token/workflow quotas before each action. Daily usage recorded in usage_records table.
  • Secrets: GCP Secret Manager with caching (TTL 300s). Per-tenant API key retrieval. Falls back to env vars.

Company package interface

Each company in src/companies/<company_id>/ exports:

  • agent_configs.py — SYSTEM_PROMPTS, AGENT_DEFINITIONS, build_registry() for all agent tiers
  • workflows.py — create_*_workflow() factory functions using TaskGraphBuilder DAG API
  • knowledge.py — seed_knowledge_base(kb) with company policies, procedures, compliance rules
  • config.yaml — Budgets, rate limits, HITL SLAs, model selections, MCP server tiers, polling intervals
  • demo.py — run_demo() for testing the full workflow end-to-end
