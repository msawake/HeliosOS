# Project Documentation Guide (HTML Documents)

The root of this project contains a variety of `.html` documents that serve as static exports of the platform's documentation, strategic plans, and system specifications. 

## How to Read These Documents
These files are self-contained static HTML pages. The most logical way to read them is to **open them in a modern web browser**.

- **For the Framework Architecture**: Start with `forgeos-docs.html` and use the top navigation bar to explore related documents.
- **For Business & Strategy**: Start with `forgeos-pitch-deck.html` or `forgeos-summary.html`.
- **For the LeadForge Implementation**: Start with `leadforge-visual.html` to see the interactive agent delegation maps.

## Logical Grouping

### 1. Official Documentation Site
These files form the primary documentation portal and are interlinked via a navigation bar.
- `forgeos-docs.html`: **Homepage & Overview** - Start here for framework concepts.
- `forgeos-docs-architecture.html`: **Technical Architecture** - Deep dive into layers and boot sequences.
- `forgeos-docs-agents.html`: **Agent Systems** - Details on execution types and lifecycles.
- `forgeos-docs-stacks.html`: **Stack Adapters** - Comparison of ForgeOS, CrewAI, ADK, and OpenClaw.
- `forgeos-docs-api.html`: **API Reference** - Documentation for the 61 FastAPI endpoints.
- `forgeos-docs-operations.html`: **Operations** - Deployment, configuration, and monitoring.

### 2. Strategic & Business Documentation
High-level strategic plans and research used during the project's conceptualization and GTM phases.
- `forgeos-pitch-deck.html`: Project presentation deck.
- `forgeos-deployment-roadmap.html`: Strategic timeline for platform rollout.
- `forgeos-gtm-research.html`: Go-to-Market research and analysis.
- `forgeos-hybrid-strategy.html`: Strategic discussion on hybrid agent execution.
- `forgeos-strategic-discussions.html`: Consolidated notes from strategic planning sessions.

### 3. Technical & System Summaries
Summaries of the platform's capabilities, infrastructure, and technical state.
- `forgeos-complete-summary.html`: Comprehensive end-to-end project summary.
- `forgeos-summary.html`: Executive summary of the platform.
- `forgeos-system-summary.html`: Technical overview of the system components.
- `forgeos-platform.html`: Platform-level architecture overview.
- `forgeos-capacity.html`: Analysis of system performance and agent capacity.
- `forgeos-infrastructure-stack.html`: Overview of the production infrastructure components.

### 4. Agent Catalogs & Concept Guides
Specifications for agents and conceptual guides on how they operate within the framework.
- `forgeos-agents.html`: Detailed catalog of 50 agents across 5 categories.
- `forgeos-candidate-agents.html`: List of proposed or candidate agents for expansion.
- `forgeos-how-agents-work.html`: Conceptual guide explaining agent mechanics.
- `types-agents.html`: Typology of agent behaviors and execution patterns.

### 5. Company-Specific Reports (LeadForge)
Documentation specifically focused on the **LeadForge** implementation (one of the 5 company variants).
- `leadforge-visual.html`: **Interactive Visualizer** - Shows agent delegation and department flows.
- `leadforge-platform.html`: Architecture overview specific to the LeadForge instance.
- `leadforge-team.html`: LeadForge organizational and department structure.
- `leadforge-day.html`: Operational walkthrough of a typical "day in the life" of LeadForge.

---

*Note: Some of these files may contain overlapping information as they were generated at different stages of the project's evolution (v1 to v3).*
