Deal AnalystKernel

BANT Score: 87/100

6
Lead Researcher delegates to Risk Assessor via A2A (cross-namespace — needs capability token)
Lead ResearcherRisk AssessorKernel
▶

The Lead Researcher needs a risk assessment from the finance namespace. Cross-namespace A2A is denied by default, so it requests a capability token first.

runtime.check_a2a("finance", "risk-assessor") → deny: cross-namespace without ACL
runtime.request_capability(target="finance/risk-assessor", verb="a2a.invoke", ttl=600) → CapabilityManager.issue() → token=cap_789...
agent__call(ns="finance", name="risk-assessor", context={capability_token: "cap_789"}) → A2AHandler: token valid → bypass ACL → allow
Capability token flow:
  1. runtime.check_a2a("finance", "risk-assessor") → deny (cross-namespace)
  2. runtime.request_capability(target="finance/risk-assessor") → token issued (10min TTL)
  3. agent__call with token → CapabilityManager.authorize() → valid → allow

Risk Assessor runs (finance namespace):
  - Checks MegaInc financial health
  - Evaluates contract risk at $2.5M
  - Data boundary: can only read finance namespace
  - Returns: risk_level="medium", notes="Standard enterprise terms"

After use: runtime.revoke_capability(token) — token deleted, access revoked.

  Deal value: $2,500,000 > max_value $1,000,000
  → Auto-delegation DOES NOT apply (over limit)
  → Request stays PENDING — David must review manually

Delivery: P0_CRITICAL priority
  DashboardChannel → urgent notification
  SlackChannel → DM to @david.park
  EmailChannel → david@company.com

Note: If the deal were under $1M, David's auto-delegation rule would have auto-approved it instantly. The $2.5M value exceeds the threshold, so David must review personally.
