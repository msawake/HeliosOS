The Key Insight

Not every agent needs frontier intelligence. A lead router that checks ZIP codes doesn't need the same model as an executive weighing a $50K budget reallocation. By matching model capability to agent complexity, you run 5x more agents at 80% less cost.

How It Works: Provider Routing in model_client.py

ForgeOS auto-detects the provider from the model name prefix and routes to the right backend:

# Model name prefix determines which backend handles the call
class ModelProvider(Enum):
    ANTHROPIC = "anthropic"  # claude-* models
    OPENAI = "openai"      # gpt-*, o3-*, o4-*
    OLLAMA = "ollama"      # ollama-*, local-*  ← NEW
    VLLM = "vllm"          # vllm-*, ft-*      ← NEW

# Ollama and vLLM both expose OpenAI-compatible APIs
# So they inherit from OpenAIClient with a different base_url
class OllamaClient(OpenAIClient):
    def __init__(self):
        url = os.environ.get("OLLAMA_BASE_URL", "http://localhost:11434/v1")
        super().__init__(api_key="ollama", base_url=url)

class VLLMClient(OpenAIClient):
    def __init__(self):
        url = os.environ.get("VLLM_BASE_URL", "http://localhost:8000/v1")
        super().__init__(api_key="vllm", base_url=url)
Zero new dependencies. Both Ollama and vLLM expose OpenAI-compatible APIs, so OllamaClient and VLLMClient inherit from the existing OpenAIClient — just with a different base URL. The openai Python package talks to all three backends.
