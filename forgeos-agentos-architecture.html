Agent Status States

IDLE → RUNNING → COMPLETED
RUNNING → FAILED → (restart_on_failure) RUNNING
FAILED → (N consecutive crashes) QUARANTINED (requires manual restart)
STOPPED PAUSED (external control via API)

Autonomous Loop

for i in range(max_iterations):    # default 50
  prompt = f"[Iteration {i}] Goal: {goal}"
  result = await invoke(agent_id, prompt)

  if result.status == COMPLETED:   # [GOAL_COMPLETE] detected
    break
  if result.status == IDLE:        # goal not yet met, continue
    continue
  if crash_count >= max_crashes:
    status = QUARANTINED           # blocks auto-recovery
    return
  backoff = min(60, 2 ** crash_count)
  await sleep(backoff)

1 The AgentOS Kernel — Policy Decision Point

src/platform/kernel.py — Every meaningful agent action passes through the kernel. Composes 6 subsystems behind a unified API.

1. AdmissionController

Validates contracts before deploy.

admit(contract) → AdmissionResult
  errors: [] ← FATAL (deploy rejected)
  warnings: [] ← non-fatal

Checks: name format, namespace, uniqueness, stack, lifecycle consistency, dependencies, tool availability

2. PermissionManager

Runtime tool + A2A ACL checks.

check_tool_call(agent_id, tool_name)
  → deny list first (precedence)
  → allow list (exact + wildcard *)
check_a2a(caller, target_ns, target_name)
  → callee's canBeCalledBy ACL

3. BudgetManager

Economic limits from manifest boundaries.

check_budget(agent_id, cost_usd)
  → per_task_usd limit
  → daily_usd limit (via UsageEnforcer)

4. PolicyEngine

Declarative rules from manifest governance.

evaluate(policy_refs, context)
  deny_if: { op: "contains",
    field: "tool_name", value: "shell" }
Operators: equals, contains, gt, in

5. DataBoundaryManager

Namespace isolation + PII policy.

check_data_access(agent_id, target_ns)
  → blocked_namespaces check
  → allowed_namespaces check
get_pii_policy(agent_id)
  → allow|detect|mask|redact|block

6. AuditRecorder

Immutable record of every decision.

_audit(action, resource_id, **details)
  → audit_log.record(...)
Actions: tool.allowed, tool.denied,
  tool.budget_denied, tool.policy_denied,
  a2a.allowed, a2a.denied, agent.admitted

Composite check_tool_call() Pipeline

1. PERMISSION
whitelist + deny list
2. BUDGET
cost + token limits
3. POLICY
JSON-logic rules
4. AUDIT
record decision
KernelDecision
allow | deny | ask_human

Short-circuits on first denial. Every decision gets an audit_id for forensic correlation. The caller (agent or SDK) only sees the final KernelDecision.
