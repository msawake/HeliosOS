Platform Architecture

Layer 4 — Intelligence

Ontology + Connectors + Intelligence Agents

Knowledge graph with typed objects & relationships. 5 industry schemas. 4 data connectors (CSV, Sheets, API, DB). 3 intelligence agents (analyst, monitor, reporter). Palantir-like business intelligence.

Layer 3 — Admin

Admin Orchestrator + Chat + Monitor

12 admin tools. Natural language control via /admin chat. Background monitoring for overdue approvals, cost anomalies, escalations. Fast-path commands (instant, no LLM needed).

Multi-Stack Agent Platform

4 agent stacks (ForgeOS, CrewAI, ADK, OpenClaw). 5 execution types (Always-On, Scheduled, Event-Driven, Reflex, Autonomous). Platform executor, registry, scheduler, event bus. 50 shared + 9 personal agents.

Intelligence Platform (Palantir-Like)

Ontology (Knowledge Graph)

Typed business objects with properties and relationships. Thread-safe with RLock. Depth-limited traversal (max 10 hops, 1000 results). Schema validation on YAML load.

5 Industry Schemas

b2b_sales (6 types, 8 links), deals (7/8), travel (6/7), insurance (6/8), real_estate (7/8). Each with Customer, Lead/Deal, and domain-specific types.

4 Data Connectors

CSV (with encoding detection), Google Sheets, REST API, Database. Sync engine with background loop. Incremental sync via SHA-256 hash tracking.

3 Intelligence Agents

intel-analyst (GPT-4o, ad-hoc questions), intel-monitor (GPT-4o-mini, anomaly detection), intel-reporter (GPT-4o, structured reports).
