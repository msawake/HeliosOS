LLM Model Routing

The LLM Router auto-detects the provider from the model name prefix:

PrefixProviderExamples
claude-*Anthropicclaude-sonnet-4-5-20250514, claude-opus-4-6
gpt-*OpenAIgpt-4o, gpt-4o-mini
o3-*, o1-*OpenAIo3-mini

Failover: set metadata.fallback_provider in LLMConfig to auto-retry on the other provider.

Database

Migrations

FileContent
001_schema.sqlCore schema: tenants, users, agents, sessions, events, approvals, audit_log
002_platform_tables.sqlPlatform: agent_registry, scheduled_jobs, event_subscriptions
003_ontology_tables.sqlKnowledge graph: entities, relationships, signals
004_client_mcp_configs.sqlPer-client MCP server configurations
005_audit_log.sqlNo-op (audit_log already in 001)

Migrations run automatically on boot. Skip with FORGEOS_SKIP_MIGRATIONS=1.

Multi-tenant RLS: All tables have tenant_id with Row-Level Security policies. The DatabaseClient.tenant(id) context manager calls set_config('app.current_tenant', id, true) at the session level.

Project History

Phase 1: AI-Operated Company Mar 8–21

Built LeadForge AI — 26 agents across 7 departments. Single-company architecture with Flask dashboard. Core: AgentInvoker, 7-check hook chain, MCP tools, event bus, HITL gateway, knowledge base.

Phase 2: Multi-Tenant SaaS Mar 21–28

Added 5 company packages. PostgreSQL with RLS multi-tenancy. Firebase auth, Stripe billing, GCP infrastructure via Terraform. Docker containerization with Cloud Build CI/CD.

Phase 3: Multi-Stack Platform Mar 28 – Apr 7

Added 4 stack adapters (ForgeOS, CrewAI, ADK, OpenClaw). Platform layer (registry, executor, scheduler, event bus, LLM router). Next.js dashboard. 50+ agents deployed. 5 execution types.

Phase 4: Production Hardening Apr 7–12

Multi-turn agent chat with SSE streaming. HITL inline approvals. Edit with AI wizard. PostgreSQL persistence via Docker. Deep audit: 28 bug fixes (boot hangs, memory leaks, race conditions, crash loops).
