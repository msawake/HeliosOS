Agent Invocation Pipeline

AgentInvoker.invoke(agent_id, task)
  → Registry lookup (AgentConfig)
  → Usage enforcer pre-flight (check_tokens)
  → HookChain.pre_tool_use() [6-hook governance]
  → ClaudeClient.run() [multi-turn agentic loop]
       → LLM call with retry (3 attempts, exponential backoff)
       → Tool execution via ToolExecutor (custom/MCP/platform)
       → Session checkpoint after each turn
       → Loop until max_turns or end_turn
  → HookChain.post_tool_use()
  → AgentResult {status, result, tokens, cost_usd, tool_calls, artifacts}

2. Core Platform Layer

agent_invoker.py

Universal entry point for all agent invocations

  • AgentTier enum: HUMAN(0) → EXECUTIVE(1) → DEPARTMENT_LEAD(2) → WORKER(3)
  • AgentConfig: agent_id, name, department, tier, system_prompt, allowed_tools, model, max_turns, timeout_seconds, budget_tokens, mcp_servers, subagents
  • AgentRegistry: register(), get(), list_by_department(), list_by_tier(), set_session_status(), get_active_sessions()
  • AgentInvoker.invoke(): Pre-flight usage check → build context → hook chain → agentic loop → result
  • delegate_to_subagent(): Retry with max_attempts, tier validation
  • delegate_parallel(): asyncio.gather() for parallel delegation
  • AgentResult: status, result, error, tokens_consumed, cost_usd, duration_seconds, tool_calls, artifacts

claude_client.py

Provider-agnostic multi-turn agentic loop

  • ClaudeClient.run(): Async entry → spawns sync loop in thread
  • Loop: LLM call → extract tools/text → execute tools → checkpoint → repeat
  • Tool timeout: 300s (configurable TOOL_EXECUTION_TIMEOUT)
  • LLM retry: 3 attempts, exponential backoff 1s→30s max
  • Retryable: ConnectionError, TimeoutError, RateLimitError, APIConnectionError
  • Session checkpointing: last_turn, last_tool_calls, partial_result saved per turn

model_client.py

Multi-provider LLM abstraction

  • LLMClient protocol: send_message() → LLMResponse
  • AnthropicClient: Claude Opus/Sonnet/Haiku
  • OpenAIClient: GPT-4o, GPT-4o-mini, o3-mini
  • OllamaClient: Local models via Ollama (OpenAI-compatible)
  • VLLMClient: Corporate A100 cluster via vLLM
  • MODEL_PRICING: 13 models with per-token input/output costs
  • Auto-detection: claude-* → Anthropic, gpt-* → OpenAI, ollama-* → Ollama, vllm-* → vLLM

hooks.py

6-hook defense-in-depth governance

  • AuditLogger: Append-only trail, PostgreSQL + buffer
  • RateLimiter: Per-agent 100/session, 30/minute
  • AuthChecker: Tool permissions by tier, dangerous pattern blocking
  • CostTracker: Budget pre-check, threading.Lock, reserve-on-pre_check
  • ComplianceChecker: Blocklist, disclaimers, email mass-send check
  • SlackNotifier: Channel routing (security, escalation, incident, approval)
  • HookChain: Orchestrator, fail-fast, ASK_HUMAN → HITL

session_store.py

Agent session persistence & checkpointing

  • AgentSession: session_id, agent_id, tenant_id, status, conversation history, cost tracking, checkpoint_data
  • InMemorySessionStore: Dev/test backend
  • PostgresSessionStore: Production with agent_sessions table
  • get_resumable(agent_id): Crash recovery — find latest running session

database.py

Multi-tenant database with RLS

  • DatabaseClient.tenant(id): Sets app.current_tenant session variable
  • PostgreSQL RLS auto-filters all queries by tenant_id
  • Connection pooling: min=2, max=10 (psycopg)
  • Supports: DATABASE_URL or CLOUD_SQL_INSTANCE (GCP)

redis_rate_limiter.py

Distributed rate limiting via Redis. Atomic INCR + EXPIRE. Falls back to in-memory when Redis unavailable.

secrets.py

GCP Secret Manager with caching + env var fallback. Per-tenant API key retrieval.

4. Agent Hierarchy (3-Tier Delegation Model)

Core Rule: Only lower-numbered tiers can delegate to higher-numbered tiers. Tier 3 workers are leaf nodes — they cannot spawn sub-agents. Delegation is validated in delegate_to_subagent().
TIER 1: EXECUTIVE OpusCEO, COO, CFO — 3 per company — 40-50 turns — Set strategy, delegate to departments, approve >$5K
TIER 2: DEPT LEAD Opus/Sonnet4-8 per company — 25-40 turns — Orchestrate department, delegate to workers, make dept-level decisions
TIER 3: WORKER Sonnet/Haiku10-18 per company — 10-40 turns — Execute specific tasks, cannot delegate, submit work upward

5. Agent Type Categories

Simple Reflex

IF/THEN rules, no state

  • Model: Local Mac 3B (Ollama/MLX)
  • Cost: $0/call, <200ms latency
  • Examples: auto-responder, lead-router, alert-agent
  • Test: Can you write it as a flowchart?

Model-Based Reflex

Pattern recognition, state tracking

  • Model: A100 fine-tuned 70B (vLLM)
  • Cost: ~$0.001/call, 2-5s latency
  • Examples: lead-scorer, fraud-detector, engagement-tracker
  • Advantage: fine-tuned on YOUR data

Goal-Based

Multi-step planning toward objectives

  • Model: A100 70B or Claude Sonnet
  • Examples: campaign-builder, search-lead, itinerary-planner
  • Has sub-goals and can sequence actions

Utility-Based

Tradeoff reasoning with constraints

  • Model: Claude Opus (required)
  • Examples: budget-optimizer, offer-drafter, counter-negotiator
  • Weighs diminishing returns, constraints

Autonomous

Self-correcting with reflection loops

  • Model: Claude Opus (required)
  • Examples: exec-ceo, exec-coo (observe-reflect-adjust)
  • Analyzes own strategy and improves

6. Scheduling Frameworks

Always-On

Continuous operation, persistent session. Runs on polling cycles.

  • Executive: 30 min cycle
  • Department: 15 min cycle
  • Dispatcher: 5 min cycle
  • Watchdog: 60 min cycle

Examples: ops-monitoring, sales-sdr, sales-nurture

Scheduled

Cron jobs via APScheduler with PostgreSQL job store for durability.

  • Standard 5-field cron
  • CronTrigger expressions

Examples: Weekly financial reporting, daily standup digest, weekly business review

Event-Triggered

Fire on specific events from the EventBus.

  • New lead, deal closed
  • Threshold breached
  • System anomaly detected

Examples: onboarding-orchestrator, incident-orchestrator, fraud-detector

On-Demand / HITL

Human-initiated via dashboard, API, or admin chat.

  • Proposal generation
  • Expense categorization
  • Contract review

SLAs: Financial 24h, Content 4h, Contract 48h, Compliance 4h

8. 6-Hook Governance Chain

Execution order is strict. Every tool call passes through all 6 hooks in sequence. First BLOCK decision halts execution. ASK_HUMAN routes to HITL Gateway for dashboard approval.
1
Audit Logger
Immutable append-only trail. SHA-256 input hashing. PostgreSQL audit_log table.
2
Rate Limiter
100 calls/session, 30/minute. Keyed by agent_id. Auto-cleanup stale agents (1h).
3
Auth Checker
Tool permissions by tier. Dangerous patterns blocked (rm -rf, DROP TABLE). T3 cannot spawn agents.
4
Cost Tracker
Pre-check reserves cost before API call. threading.Lock. $50/session default. Daily token tracking.
5
Compliance
Blocklist (guarantees, PII). Disclaimers (financial, legal, medical). Email mass-send check (>50 → HITL).
6
Slack Notify
Channel routing: #security, #escalation, #incidents, #approvals. Pending notification buffer.

14. Security Features

Authentication & Authorization

  • Firebase JWT (RS256) with RBAC
  • API key auth with hmac.compare_digest (timing-safe)
  • Per-IP rate limiting: 10 failures → 5-min block
  • 3 roles: Admin (full), Operator (approve/monitor), Viewer (read-only)

Data Isolation

  • PostgreSQL Row-Level Security on ALL 18 tables
  • app.current_tenant session variable per request
  • No cross-tenant data exposure at database layer
  • Tenant-scoped API keys and Stripe customers

Governance Controls

  • 6-hook chain on every tool call
