The Core Concept: Framework vs Agents

The ForgeOS Framework

The platform. The operating system. Boots once, runs forever.

  • Agent Registry — stores and queries all agents
  • Platform Executor — deploys, invokes, recovers agents
  • LLM Router — routes to Anthropic/OpenAI with retry + failover
  • Agentic Loop — LLM ↔ tool execution cycle
  • Scheduler — cron-based agent scheduling
  • Event Bus — pub/sub for cross-agent messaging
  • Tool Executor — MCP + custom tool routing
  • Dashboard — Next.js admin console
  • FastAPI — 61 REST endpoints
  • Audit + Metrics + Alerts

The Agents

The workloads. The programs. Deployed, invoked, managed by the framework.

  • Defined by AgentDefinition
  • Name, stack, execution type, tools, system prompt
  • 5 execution lifecycles (always-on, scheduled, event, reflex, autonomous)
  • 3 ownership types (personal, shared, client)
  • 4 stack adapters (ForgeOS, CrewAI, ADK, OpenClaw)
  • Deploy via API, dashboard, or AI wizard
  • Invoke via REST API or streaming chat
  • 74 agents across sales, marketing, finance, ops, HR, legal
  • Each has a system prompt, tool whitelist, and LLM config
  • 7 status states (idle, running, paused, stopped, completed, failed, quarantined)
Think of it like Kubernetes: ForgeOS is the cluster — it provides scheduling, networking (event bus), storage (session store), and monitoring (audit log). Agents are the pods — you define what they do, and the framework runs them. The framework does not perform business tasks; agents do.

Quick Start

# 1. Install
pip install -e ".[dev]"

# 2. Configure
echo 'ANTHROPIC_API_KEY=sk-ant-...' > .env

# 3. Boot
PYTHONPATH=. python3 -m src.bootstrap --no-auth --dashboard --port 5000

# 4. Dashboard (separate terminal)
cd dashboard && npm install && npm run dev

# 5. Deploy first agent
curl -s -X POST http://localhost:5000/api/platform/agents \
  -H "Content-Type: application/json" \
  -d '{"name":"hello-agent","stack":"forgeos","execution_type":"reflex",
       "description":"Test agent","chat_model":"claude-sonnet-4-5-20250514"}'
