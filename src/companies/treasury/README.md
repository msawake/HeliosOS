# Treasury company pack (mock-data prototype)

Phase-1 reconciliation agents for the Treasury department, built on Helios OS.
Because access to the finance BigQuery warehouse is still pending, the agents
run against **synthetic CSV data** generated by `mock_data.py`. The data source
is the *only* thing that changes when real access lands — agent prompts, tools,
and logic stay identical.

## Agents (`agents/*.yaml`)

| Agent | Lifecycle | What it does |
|---|---|---|
| `kyriba-chat-orchestrator` | reflex (invoke on demand) | Conversational front door; routes a request to the right reconciler via A2A and summarizes. |
| `bank-sap-reconciliation` | scheduled (07:00 wkdays) | Matches bank inflows ↔ SAP open AR items; classifies exceptions; escalates. |
| `debt-reconciliation` | scheduled (08:00 wkdays) | Scheduled debt service vs actual payments; missed/over payments. |
| `po-reconciliation` | scheduled (08:00 wkdays) | 3-way match PO ↔ invoice ↔ payment; over-bill, duplicates, maverick spend. |
| `mapping-classification` | scheduled (06:00) + A2A service | Resolves counterparties/vendors to SAP ids + GL accounts; audits mapping gaps. |

All agents are **read-only analysts**: they produce reports and exception lists
and escalate material items to the approver (`antoni.bergas@makingscience.com`)
via `human__ask`. None post to SAP, release payments, or send external email.

## Mock data (`mock_data.py` → `data/*.csv`)

13 internally-consistent tables, as-of 2026-06-09, EUR. Each contains clean
matches **and** planted exceptions so the agents have real work:

- **Bank↔SAP:** clean matches, a short-payment (fee), an overpayment, an
  unmapped counterparty (`ZENITH HOLDINGS`), an unidentified receipt
  (`PAYPAL EUROPE`), an alias match, and a duplicate.
- **Debt:** a clean payment, a missed principal, an overpayment, a payment on an
  unknown loan.
- **PO:** a clean 3-way match, a PO over-bill, a duplicate invoice, maverick
  (no-PO) spend, an open commitment.
- **Cash quality:** Kyriba vs BigQuery mirror with one balance drift and one
  account missing from the mirror.

Regenerate: `python -m src.companies.treasury.mock_data` (deterministic).

## Run it

Each agent has its **own Google service account** and reads/writes its data in a
**Google Drive folder shared with that SA** (`drive__*` tools), not the local
filesystem. The full walkthrough — per-agent SA identity, the kernel-gated Drive
write approval, and the chat demo script — is in [`docs/treasury-demo.md`](../../../docs/treasury-demo.md).

```bash
# 1. (re)generate the synthetic CSVs (the data you upload to each agent's folder)
PYTHONPATH=. .venv/bin/python3 -m src.companies.treasury.mock_data

# 2. one-time: provision the 5 per-agent SAs (idempotent)
for s in bank-sap debt po mapping kyriba; do ./scripts/provision_agent_sa.sh "$s"; done

# 3. one-time: in a Shared Drive, create a folder per data agent, share it with
#    that agent's SA email, and upload its CSVs (see the table in docs/treasury-demo.md),
#    then write the folder ids into the manifests:
./scripts/set_treasury_folders.sh bank-sap=<ID> debt=<ID> po=<ID> mapping=<ID>

# 4. boot the platform with this company pack
PYTHONPATH=. .venv/bin/python3 -m src.bootstrap --company treasury --dashboard --port 5000

# 5. deploy the agents (CLI or MCP forgeos_deploy_yaml)
for a in kyriba-chat-orchestrator bank-sap-reconciliation debt-reconciliation \
         po-reconciliation mapping-classification; do
  forgeos deploy src/companies/treasury/agents/$a.yaml
done

# 6. drive it — chat with the orchestrator (it routes to specialists via A2A)
forgeos invoke kyriba-chat-orchestrator "Reconcile today's bank inflows against SAP." --wait
```

Each specialist reads its input CSVs from its Drive folder with `drive__read_file`
and writes its report + exceptions back with `drive__create_file` — and every
Drive **write pauses for human approval** (kernel `ask_human`).

## Mock → BigQuery transfer

When the finance project + `roles/bigquery.dataViewer` are granted:

1. Set `mcp_servers.tier1.bigquery.required: true` in `config.yaml` (or keep
   shell `bq`, already allowlisted).
2. In each manifest, flip `metadata.data_source: bigquery`.
3. The agents swap each `drive__read_file` of `<table>.csv` for the equivalent
   `bq query --use_legacy_sql=false '...'`. No prompt or logic change.
